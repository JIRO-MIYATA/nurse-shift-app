# 勤務人数検証・修正ループの実装結果 (Walkthrough)

## 変更内容
ナースシフト管理アプリのスケジュール自動生成機能において、設定された各種勤務人数（日勤・準夜勤・深夜勤）の制約を厳密に満たすようにロジックを修正しました。

`src/components/NurseShiftApp.jsx` 内にある `generateSchedule` 関数の `[PHASE 8]`（最終調整処理）を以下のように新しく書き換えました：
- **`while` ループによる検証・修正 (Phase 9相当)**
  最大50回のループ内で、「START (準夜勤) の人数」、「DEEP (深夜勤) の人数」、および「DAY (日勤) の人数」をそれぞれチェックし、過不足があれば自動で追加または削除を行います。
- **制約パターンの維持**
  「準夜勤 → 深夜勤 → 休み」という一連のシフトパターンを満たした場合に入る強制的な「休」が、人数調整ロジックによって誤って「日勤」に上書きされないように保護（`isPatternOff` チェック）しています。
- **連勤制限の厳格化**
  人数調整によって「休」を「日勤」等に変更する際、設定された最大連勤数（6連勤以上不可）を検知する機能（`wouldExceedMaxConsecutive`）を追加しました。さらに、万が一6連勤以上の箇所が発生した場合は自動的に連勤中の「日勤」を優先して「休」に変更し、再度ループで人数を合わせる仕組みを組み込んでいます。
- 全ての日で設定人数とシフト数が完全に一致し、連勤制限違反がない場合（`isPerfect = true`）に初めてループを抜け、スケジュールとして確定します。万が一パズルのように一致不可能な場合も、50回の最大試行で無限ループを防ぐ安全設計にしています。

## 追加修正：リーダー割当のタイミング修正
「休日の人にリーダーが割り当てられてしまう」問題に対応するため、休日の判定が最後まで確定した後の処理（全ループ終了後）でリーダー選出の処理を実行するよう修正しました。
