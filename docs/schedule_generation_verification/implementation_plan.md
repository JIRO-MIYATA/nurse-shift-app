# 勤務人数検証・修正ループの実装計画

## 目標
シフト生成機能において、設定された勤務人数（日勤、準夜勤、深夜勤）の制約を満たさずに生成が終了してしまう問題を解消する。
生成後に各シフトの人数が正しいか再度確認し、一致するまで（または上限回数まで）過不足を自動で修正するループ処理を実装する。

## 確認事項
現状の「PHASE 8」では左からの順次処理で人数を強制的に合わせていますが、修正によって他の日の条件（「準夜→深夜→休」のパターンなど）が崩れてしまい、結果として人数の過不足が生じたままになっています。
今回、PHASE 8を「最終検証および修正ループ」に置き換え、全シフト（START, DEEP, DAY）の人数とパターンの整合性が完全に一致するまで繰り返し修正を行うようにします。

## 修正内容

### `src/components/NurseShiftApp.jsx`
- **[MODIFY]** `generateSchedule` 関数内の `[PHASE 8]` セクション全体を書き換えます。
- **追加するロジック:**
  - `let isPerfect = false;` と `let loopCount = 0;` を使用した `while` ループを導入（最大50回程度で無限ループを防止）。
  - ループ内で以下を順に実行：
    1. **START人数の確認・修正:** 不足時は追加（追加に伴い翌日のDEEP、翌々日のOFFも仮決定）。過剰時はDAYに戻す（それに伴う翌日のDEEPもDAYに戻す）。
    2. **DEEP人数の確認・修正:** 前日がSTARTであるスタッフを優先してDEEPを割り当て、過不足を調整。
    3. **パターンの強制サニタイズ:** 全スタッフについて、「START -> DEEP -> OFF」の並びになっていない箇所があれば強制的に修正（パターンの不整合があれば `isPerfect = false` とし、再度ループさせる）。
    4. **DAY人数の確認・修正:** 祝日・平日の設定人数に合わせて調整。この際、**「START -> DEEP -> OFF」によって設定されたOFF（休）をDAY（日）に上書きしないように保護**する。
- 全ての条件に違反がなくなり `isPerfect === true` となった時点でループを終了し、スケジュールとして確定します。

## 検証計画
### 手動検証
1. ブラウザを起動して、アプリケーション画面を開く（`npm run dev`）。
2. 「スケジュール自動生成」ボタンを押下して数ヶ月分のスケジュールを生成してみる。
3. 生成後、画面下部やセル内の警告色（赤やオレンジ）が出ていないか（＝勤務人数の設定が厳密に守られているか）を目視確認する。
4. シフトパターン「準夜→深夜→休」が含まれる箇所を探し、そのパターンが崩れずに維持されていることを確認する。
