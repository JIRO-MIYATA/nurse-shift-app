# 看護師シフト管理システム - ウォークスルー

シングルファイルReactコンポーネント `NurseShiftApp.jsx` として実装された、看護師シフト管理システムの概要と検証方法です。

## 変更点

### 1. リーダー機能 (Dynamic Leader Assignment)
-   **仕様**:
    -   リーダーは「スタッフの属性」ではなく、**「日ごとの担当」**として管理します。
    -   シフト表のセルをクリックし、編集モーダル内の「👑 リーダー任命/解除」ボタンで切り替えます。
    -   リーダー担当セルには王冠マーク（👑）が表示されます。
    -   **制約**: 新人（🔰）はリーダーに任命できません（ボタンが無効化されます）。
    -   **警告**: 準夜・深夜の列にリーダーが一人もいない場合、列ヘッダーがマゼンタ色で警告されます。
    -   **初期値**: **新人（🔰）以外のスタッフは、デフォルトでリーダー属性ON**として初期化されます。

### 2. 夜勤回数の平準化 (Equalization & Drop-off Fix) - <span style="color:orange">改善</span>
-   **仕様**:
    -   夜勤回数が少ないスタッフを優先しつつ、適度なランダム性を持たせて割り当てます。
    -   **修正**: 夜勤候補を選ぶ際、リーダー属性を持つスタッフが候補から外れてしまい、夜勤が組めなくなる（0人になる）不具合を修正しました。必ずリーダーを候補に含めることで、安定して夜勤を割り当てます。

### 3. 連勤制限の強化 (Consecutive Work Limit Fix) - <span style="color:orange">改善</span>
-   **仕様**:
    -   最大5連勤とし、6日目は必ず「休み（休）」にします。
    -   **改善点**: 6連勤目となる日が「夜勤入り」などで変更できない場合、**その前日を休み**に変更することで、確実に連勤を回避するようにロジックを強化しました。
    -   **前月考慮**: 前月の勤務情報もカウントに含めるようロジックを修正しました。例えば、前月末に3日間勤務していた場合、当月は2日働いた時点で5連勤とみなされ、3日目は休みになります。

### 4. UI改善 (UI Refinements)
-   **休日数の表示**: テーブルの右端に「休日」列を追加し、月間の休日数（「休」の数）を表示しました。
-   **夜勤固定の強調**: 「夜勤回数固定」の設定があるスタッフ（例: スタッフ1など）の準夜・深夜回数を **水色かつ太字** で表示し、制限がわかりやすくなるようにしました。
-   **希望アイコン**: 希望タイプに応じて「日」（日勤希望）、「休」（休み希望）と表示するようにしました。
-   **編集モーダルからの希望設定**: シフト表のセルをクリックして出る編集画面から、直接「休み希望」「日勤希望」を設定できるようになりました。

### 5. 日勤回数の平準化 (Day Shift Equalization)
-   **仕様**:
    -   平日の日勤人数制限を守るために休みを入れる際、**その時点での日勤数が一番多い人** を優先的に選びます。

### 6. リーダー属性と自動配置 (Leader Attribute & Constraints)
-   **仕様**:
    -   スタッフ管理画面に「リーダー」チェックボックスを追加しました。
    -   シフト生成時、**日勤・準夜・深夜の各グループに、必ず1名以上のリーダー属性持ちスタッフが含まれる**ように制御されます。
    
### 7. 日勤希望 (Day Shift Requests) - <span style="color:orange">改善</span>
-   **仕様**:
    -   希望入力画面で、セルをクリックすると「なし」→「休（オレンジ）」→「日（青）」→「なし」と切り替わります。
    -   「日」を選択して作成すると、その日は必ず「日勤」になります（夜勤にはなりません）。
    -   **修正**: 「日」希望を出したにも関わらず、夜勤が入ってしまう不具合を修正しました。

### 8. データの保存・読込 (Export/Import) - <span style="color:cyan">新規</span>
-   **仕様**:
    -   画面上部の「💾 保存」ボタンで、現在のスタッフ情報・希望情報をJSONファイルとして保存できます。
    -   「📂 読込」ボタンで保存したファイルを読み込み、データを復元できます。これにより、ページを更新しても以前の状態から作業を再開できます。

### 9. 前月の勤務情報設定 (Previous Month Data) - <span style="color:cyan">新規</span>
-   **仕様**:
    -   画面上部の「⏮ 前月設定」ボタンから、**前月末7日間の勤務状態**を登録できます。
    -   **準夜→深夜の連携**: 前月の最終日が「準夜勤」だった場合、当月の1日目は自動的に「深夜勤」に設定されます。そして2日目は「休み」が推奨されます。
    -   **深夜→休の連携**: 前月の最終日が「深夜勤」だった場合、当月の1日目は「休み」に設定されます。
    -   **キャッシュ機能**: 月を移動（例: 4月→5月→4月）しても、入力した前月設定や希望は保持されます。

## 検証手順

1.  **前月設定の連携確認**:
    -   4月の前月設定（3/31）を「準」にします。
    -   4月のシフトを作成します。
    -   4/1が「深」になっていることを確認します。
2.  **前月からの連勤確認**:
    -   4月の前月設定を5日間「日」にします。
    -   4月のシフトを作成します。
    -   4/1が「休」になっていることを確認します。

## 追加実装の検証結果 (Latest)

### 10. UI改善 (UI Refinements)
-   **ヘッダー配色の変更**:
    -   土曜日: 青色の背景
    -   日曜/祝日: 赤色の背景
    -   平日: 通常の濃紺/グレー
-   **スタッフ番号**:
    -   管理IDではなく、表示順に「1, 2, 3...」と連番で表示するように変更。削除しても番号が飛びません。
-   **スタッフ管理モーダル**:
    -   見やすさ向上のため、テーブルに縦線（区切り線）を追加しました。

### 11. ロジック改善 (Holiday Equalization)
-   **休日数の平準化**:
    -   日勤人数の調整（Phase 4）において、休みを入れるスタッフを選ぶ際、**「総勤務日数が最も多い（休日が少ない）スタッフ」**を優先的に選ぶように重み付けを変更しました。
    -   これにより、チーム全体の休日数がより均等になるように調整されます。

### 12. 前月データの自動連携・範囲拡大 (Auto-link & 10-day Expansion)
-   **自動連携機能**:
    -   前月のデータがシステム内に保存されている場合、自動的にそのデータを引き継ぎます。
    -   自動連携中は「⏮ 前月設定」ボタンが「🔗 前月連携中」に変わり、手動変更が無効化されます。これにより不整合な入力が防がれます。
-   **10日間の考慮**:
    -   前月のデータを参照する範囲を「7日間」から「10日間」に拡大しました。
    -   これにより、月末から月初にかけての連勤（例えば月末に4連勤していた場合など）の判定がより確実になります。

### 13. 月末・月初の夜勤連携修正 (EOM Night Shift Fix)
-   **不具合修正**:
    -   2月の月末に「準夜」が入っても、3月1日が「深夜」にならない（2月の月末に準夜が入らない）問題を修正しました。
    -   **原因**: 月末日において、翌日・翌々日のロック判定が現在月の範囲外を参照してしまうため、安全策として割り当てがスキップされていた可能性がありました。
    -   **対応**: 月末日であっても、翌月分のスケジュール枠（範囲外）への書き込み判定を無視して、当日の「準夜」を確定させるようにロジックを緩和しました。これにより、2月末の準夜が正しく保存され、3月1日の自動連携で「深夜」が設定されます。

### 14. 月移動時のキャッシュ更新修正 (Cache Update Fix)
-   **不具合修正**:
    -   月を移動した直後（例: 2月→3月）、最新の2月データが反映されず、3月1日の連携がうまくいかない場合がある問題を修正しました。
    -   **原因**: 月移動処理とデータの読み込み処理のタイミングにより、一瞬古いデータ（またはデータなし）を参照してしまうことがありました。
    -   **対応**: データの読み込み処理（`useEffect`）が、キャッシュの更新を確実に検知するように依存関係を修正しました。

### 15. シフト生成ロジックの依存関係修正 (Dependency Fix)
-   **不具合修正**:
    -   前月のデータが画面上は読み込まれているように見えても（デバッグ表示など）、実際に「作成」ボタンを押すと連携が反映されない問題を修正しました。
    -   **原因**: シフト生成関数（`generateSchedule`）が、更新された「前月データ」を認識せず、古い状態（データなし）のまま計算を行っていたためです（プログラム上の依存関係設定漏れ）。
    -   **対応**: シフト生成関数が常に最新の「前月データ」を使用して計算するように修正しました。これにて、月移動直後でも確実に連携が動作します。

### 16. 月移動時の前月データ同期修正 (Prev Data Sync Fix)
-   **不具合修正**:
    -   一度3月を表示した後に2月のデータを変更し、再度3月を表示した場合、変更前の古い2月データ（または手動モード）が残り続ける問題を修正しました。
    -   **原因**: 月ごとのキャッシュ保存時に「その時点での前月設定」も保存してしまっており、再訪問時にキャッシュからの復元が優先され、最新の2月データを見に行っていませんでした。
    -   **対応**: 月を移動した際、キャッシュの復元よりも「最新の前月データの取得」を常に優先するようにロジックを変更しました。これにより、2月のデータを修正すれば、3月の設定も自動的に上書き更新されます。

### 17. シフト生成時の前月データ参照強化 (Robust Prev Data Lookup)
-   **不具合修正**:
    -   前月の修正がどうしても「作成」ボタンに反映されない場合がある問題（状態更新の遅延など）に対し、対策を強化しました。
    -   **対応**: シフト生成ロジック内で、画面上の表示用データ（state）だけでなく、内部データベース（monthlyCache）を直接参照して前月の最終日を確認する処理を追加しました。これにより、画面表示の更新タイミングに依存せず、確実に正しいデータで計算が行われます。

### 18. ヘッダー色（土曜日）の視認性向上 (Header Color Visibility)
-   **調整**:
    -   「土曜日の色が平日と同じに見える」というご指摘と、「土日が全て祝日（赤）になっている」という点に対応しました。
    -   **修正**: 休日判定ロジック（`checkIsHoliday`）から土曜日を除外しました。これにより、土曜日は「青色」、日曜日と祝日は「赤色」と明確に区別されます。
    -   さらに、ヘッダーだけでなく**表全体の列（ボディ・フッター）**にも背景色（土曜は薄い青、日祝は薄い赤）を適用し、視認性を大幅に向上させました。
    -   また、ホバー時のデバッグ表示も削除し、シンプルにしました。

### 19. スタッフ条件のみのインポート・エクスポート機能 (Staff Condition I/O)
-   **機能追加**:
    -   「条件読込」ボタンに加え、**「条件保存」ボタン**を追加しました。
    -   **条件保存**: 現在のスタッフ情報（名前、チーム、設定など）のみをJSONファイルとしてダウンロードします。
    -   **条件読込**: 保存したJSONファイルからスタッフ情報のみを読み込み、現在のシフト表に適用します。
    -   これにより、月が変わっても「条件保存」→「新しい月で条件読込」というフローで、スタッフ設定を簡単に引き継げるようになりました。

-   **レイアウト変更**:

        1.  **💾 データ保存** (全体バックアップ)
        2.  **📂 データ読込** (復元)
        3.  **スタッフ保存** (条件のみ)
        4.  **スタッフ読込** (条件のみ)
    -   **削除**: 「前月設定」ボタンを削除し、画面をスッキリさせました（自動連携が機能しているため）。
    -   **デザイン統一**: アイコン（💾や📂）を削除し、ボタンのサイズ（幅）を統一して整列させました。
    -   **名称変更**: 「スタッフ」→「スタッフ編集」、「希望」→「希望勤務」に変更し、アイコン（👥や📅）も削除しました。
    -   **表示修正**: 「6日から末日まで赤線が出る（連勤警告）」というご指摘に対し、警告枠の色を**赤から黒**に変更しました。
    -   **表示修正**: 「6日から末日まで赤線が出る（連勤警告）」というご指摘に対し、警告枠の色を**赤から黒**に変更しました。
    -   **表示修正**: 「6日から末日まで赤線が出る（連勤警告）」というご指摘に対し、警告枠の色を**赤から黒**に変更しました。
    -   **表示修正**: 黒い線が出ているセルと出ていないセルで「微妙なズレ」が発生していたため、以下のように修正しました：

        2.  `outline`（輪郭線）はレイアウトのスペースを一切占有しないため、物理的なズレが理論上発生しません。
        3.  さらに、警告がないセルにも「透明なoutline」を設定し、描画条件を完全に一致させました。
