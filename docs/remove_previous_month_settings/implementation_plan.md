# 実装計画: 前月設定機能の削除および自動ルールの強化

前月設定（手動入力）ボタンとそれに関連するモーダルを削除し、プログラム内部のキャッシュデータ（`monthlyCache`）またはインポートされたデータに基づいて前月の勤務実績を自動的に反映するように変更します。また、前月からの繰り越し勤務（夜勤の START -> DEEP -> OFF の流れ）および連勤制限（5連勤禁止）が正しく適用されることを保証します。

## 検証計画

### 手動での検証

- **UI の確認**:
    - アプリケーションを起動し、ヘッダーおよび凡例部分から「前月設定」ボタンが消えていることを確認します。
- **ルールの適用確認**:
    1.  現在の月のシフトを作成します（例: 3月）。
    2.  翌月（例: 4月）に移動します。
    3.  3月の最終日が「START（準夜）」だった場合、4月の初日が「DEEP（深夜）」、2日が「OFF（休）」として自動的にロックされていることを確認します（夜勤ルールの検証）。
    4.  3月の最終日から数日間連続して勤務していた場合、4月の最初の方で5連勤制限が正しく働き、必要に応じて休日が挿入されることを確認します（連勤制限の検証）。
- **データの保存・読込**:
    - データを JSON で保存し、再度読み込んだ後も前月データとの連携が正しく維持されることを確認します。
