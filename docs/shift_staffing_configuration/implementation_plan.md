# 実装計画: 勤務人数設定変更機能の追加

平日・休日それぞれの「日勤」および「準夜・深夜」の必要人数を設定画面から変更できるようにします。これまで定数（`T_DAY_WEEK`, `T_DAY_HOL`, `T_NIGHT`）として固定されていた値を React の `useState` で制御するように変更し、UI から動的に更新可能にします。

## 変更内容

### NurseShiftApp (UI)

#### [MODIFY] [NurseShiftApp.jsx](file:///c:/Users/miyat/OneDrive/Desktop/AGT-No1/src/components/NurseShiftApp.jsx)

- **状態の追加**:
    - `shiftConfig`: `{ dayWeek: 11, dayHol: 7, night: 4 }` という形式のオブジェクトで勤務人数の設定を保持します。
    - `showConfigModal`: 設定用モーダルの表示フラグです。
- **定数からステートへの移行**:
    - コード内で `T_DAY_WEEK`, `T_DAY_HOL`, `T_NIGHT` を参照している箇所を、`shiftConfig.dayWeek` などのステート参照に置き換えます。
- **JSX の修正**:
    - ヘッダーの「スタッフ編集」の隣などに「勤務人数設定」ボタンを追加します。
    - 設定値を一覧・変更できるモーダルを実装します。
    - 凡例（Legend）の下にある「日勤上限」などの表示を、現在の設定値に基づいて動的に反映するようにします。
- **ロジックの修正**:
    - `generateSchedule` および `validation` (useMemo) で、新しいステート値を参照するように修正します。
    - **バリデーションの動的反映**: ユーザーが設定した人数（新基準値）と実際の勤務人数が異なる場合、以下のルールで警告を表示します：
        - **日勤**: 基準値を超過した場合は「赤」、基準値を下回った（1名でも少ない）場合は「オレンジ」を表示するようにロジックを厳格化します。
        - **夜勤**: 基準値と一致しない場合は「赤」、新人配置やリーダー不在などの二次条件で「オレンジ」や「マゼンタ」を表示します。
- **データの保存・読込**:
    - `exportData` および `importData` に、`shiftConfig` の値も含めるようにし、永続化に対応します。

- **マニュアル表示機能の追加**:
    - ヘッダーに「ヘルプ / マニュアル」ボタンを追加します。
    - マニュアルの内容をコンポーネント内の定数または JSX として定義し、専用のモーダルで表示できるようにします。

- **リーダー自動割り当て機能の追加**:
    - `generateSchedule` の最後に、各日の「日勤(DAY)」「準夜(START)」「深夜(DEEP)」の各シフトにおいて、勤務スタッフの中からそれぞれ1名をリーダーとして選出し、`leaderFlags` に追加する処理を実装します。
    - 選出優先順位: 
        1. 既に `isLeader` 属性（スタッフ編集画面の設定）を持っているスタッフ
        2. `isLeader` がいない場合は「新人(rookie)」ではないスタッフ
        3. それ以外（全員新人の場合など）は該当シフトの先頭スタッフ

## 検証計画

### 手動での検証

- **UI の確認**:
    - 「勤務人数設定」ボタンをクリックしてモーダルが表示されることを確認。
    - 設定値を変更し、「保存」をクリックすると凡例などの表示が更新されることを確認。
- **ルールの適用確認**:
    - 夜勤人数を 4名から 3名に変更し、「✨ 作成」をクリック。各日の夜勤人数が 3名になることを確認。
    - 平日の日勤人数を 11名から 12名に変更し、自動作成後のバリデーションでオレンジや赤の警告が出ないことを確認。
- **データの保存・読込**:
    - カスタムな勤務人数設定を行った後にデータを JSON 保存し、再読込した際に設定値が維持されていることを確認。
- **リーダー自動割り当ての確認**:
    - 複数のスタッフがいる状態で「✨ 作成」を実行し、毎日「日勤」「準夜」「深夜」のそれぞれに必ず1名ずつ 👑 マークがついていることを確認。
    - 特定のシフトにリーダー属性 (`isLeader`) を持つスタッフがいる場合、そのスタッフが優先的に 👑 割り当てされていることを確認。
