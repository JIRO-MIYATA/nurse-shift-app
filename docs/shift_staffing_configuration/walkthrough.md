# 修正内容の確認: 前月設定機能の削除および自動ルールの強化

前月設定（手動入力）ボタンおよびモーダルを削除し、プログラム内部のキャッシュデータ（`monthlyCache`）に基づいた自動連携を強化しました。

## 変更内容の概要

### 勤務人数設定変更機能の追加
- ヘッダーに「勤務人数設定」ボタンを追加しました。
- 平日日勤、休日日勤、および準夜・深夜の各必要人数を個別に設定できるモーダルを実装しました。
- 設定を変更すると、スケジュール自動作成ロジックおよびバリデーション警告（赤・オレンジ）が新しい基準値に基づいて即座に動作するようになります。
- **日勤バリデーションの強化**: 日勤の人数が設定した基準値に満たない場合（1名でも少ない場合）、即座に「オレンジ」の警告が表示されるように仕様を厳格化しました。
- **操作マニュアルのシステム内閲覧機能**: ヘッダーに「ヘルプ」ボタンを追加し、アプリケーション内で詳細なマニュアルを直接確認できるモーダルを実装しました。
- この設定値は JSON 形式でのデータ保存・読込にも対応しています。

### 自動ルールの維持と適用確認
- **夜勤ルール**: 前月の最終日が「準夜（START）」または「深夜（DEEP）」だった場合、翌月の初日に「深夜（DEEP）」や「休日（OFF）」が自動的に割り当てられるロジック（Phase 0）を維持しています。
- **5連勤禁止ルール**: 前月の末尾からの連続勤務日数をカウントし、翌月の最初の方で制限（最大5連勤）を超えないように休日を挿入するロジック（Phase 5）を維持しています。
